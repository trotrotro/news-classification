# Многоклассовая классификация текстов на русском языке
## Апология

Заявленной темой моего проекта была "многоклассовая классификация текстов на осетинском языке". 
К сожалению, при выборе датасета я не учла, что может возникнуть сложность с доступом к датасету OssetianNLPWorld/ossetian-news-multiclass с открытой лицензией (CC BY 4.0). 
Я отправила заявку на доступ к датасету и написала его автору в соцсети, но к сожалению не получила ответа на просьбу предоставить мне доступ...
Поэтому я приняла единственное возможное в такой ситуации решение -- взять похожий датасет и использовать в модели только те методы, которые можно было бы
легко перенести на другой язык. 

Изначальный датасет -- это набор новостей на осетинском языке с указанием тематики, размер около 18 мб (13к записей), классы по тематикам несбалансированные.
Датасет, который я использую в работе -- новости на русском языке, размер порядка 12 мб (тоже 13к записей), классы тоже несбалансированные.

## Данные

Итак, в работе был использован датасет Russian News 2020 (https://www.kaggle.com/datasets/vfomenko/russian-news-2020), очищенный от записей, не содержащих информацию о рубрике.
Средняя длина одной записи - 1340 символов. Всего около 13k записей.
Одна запись -- это текст новостной статьи и её рубрика (одна из 15 возможных)
Данные я стратифицированно разбила на часть для обучения, валидации и проверки в соотношении 70/15/15.

## Выходные данные

Модель возвращает предсказанную рубрику для новостной статьи.

## Метрики

Для оценки качества в работе использованы метрики macro f1 и взвешенная f1, так как классы несбалансированные. Кроме них отслеживается метрика accuracy.

## Моделирование

В проекте используется трёхслойная BiLSTM с механизмом внимания и embedding-слоем.

## Бейзлайн

В качестве бейзлайна я взяла модель многоклассовой классификации на алгоритме логистической регрессии. Векторизация текста производилась методом TF-IDF. 
С результатом использования этой модели можно ознакомиться в файле notebooks/baseline.ipynb. 
Надо сказать, что в результате работы не удалось превзойти результат, который даёт бейзлайн и, более того, даже приблизиться к нему. 
Таким образом, задача поиска способа улучшения результата классификации по сравнению с реализацией на TF-IDF векторах, не учитывающих конекст, не была выполнена в этом проекте.

# Работа с проектом

## Настройка

1. Клонируйте репозиторий

```
git clone https://github.com/trotrotro/news-classification.git
cd news-classification
```

2. Установите uv и создайте виртуальную среду

```
curl -LsSf https://astral.sh/uv/install.sh | sh
uv sync
```

## Хранение данных

Данные хранятся локально через DVC и на гугл диске. При запуске обучения или инференса авотматически проверяется, можно ли получить данные через dvc pull. Если нет, то они скачиваются из гугл диска.

- Данные для обучения, валидации и проверки: https://drive.google.com/file/d/1P400Hq4axtPHkoRhZ11MhkE1CyIem_kS/view?usp=drive_link
- Чекпоинт обученной модели: https://drive.google.com/drive/folders/1b5JlmPFs-gVtWv1fJlf9jrujeC1ilkTI?usp=drive_link
- Модель в формате onnx: https://drive.google.com/drive/folders/1r8vM9tOyvVmafe2_Lt7AS_KY_cX9TFx0?usp=drive_link
- Артефакты модели: https://drive.google.com/file/d/1LEXQKJcB4vitXf-keMld-d1nxBXwSNbX/view?usp=drive_link

## Обучение модели

Для обучение нужно в одном окне терминала запустить сервер mlflow:

```
uv run mlflow server \
  --host 127.0.0.1 \
  --port 8080 \
  --backend-store-uri sqlite:///mlflow.db \
  --default-artifact-root ./mlruns
```

В другом окне запустить собственно обучение:

```
uv run python -m news_classifier.train
```

По завершении обучения чекпоинт лучшей модели сохраяняется в директорию models, артефакты модели сохраняются в директорию artefacts.

## Логируемые параметры

По адресу http://127.0.0.1:8080/ после обучения можно найти графики логируемых параметров, важнейшие из которых - accuracy, f1 и значение функции потерь на данных для валидации. Графики для сохраненной модели можно найти в директории plots.

## Инференс

Проект поддерживает два режима инференса

1. Предсказания для текстов в csv-файле. Запуск этого режима:

```
uv run python -m news_classifier.infer
```

Результат работы: предсказания сохраняеются в файл predictions.csv, выводятся 5 первых предсказаний.

2. Предсказание для введённого текста. Запуск режима:

```
uv run python -m news_classifier.infer infer.interactive=true
```

После запуска появится сообщение "Введите текст новости:". После введения и отправки текста будет выведено предсказание рубрики для этого текста.

## Экспорт в onnx

Для экспорта модели нужно выполнить:

```
uv run python -m news_classifier.deployment.export_onnx
```

Модель будет кспортирована в файл deployment/model.onnx
